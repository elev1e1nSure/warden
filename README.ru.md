> **Опиши задачу. Не пиши скрипт.**

Warden — AI-агент, который живёт в твоём терминале и управляет твоим Windows-компьютером. Говоришь что нужно сделать — он сам разбирается как, выполняет и показывает каждый шаг в реальном времени.

Видит экран, запускает команды, редактирует файлы, водит браузер. Никаких плагинов для IDE. Никакого Electron. Не нужен аккаунт в облаке.

## Установка

[**⬇ Скачать для Windows x64**](https://github.com/elev1e1nSure/warden/releases/latest)

Разархивируй куда угодно. Два бинарника, одна папка, ноль настройки. Запусти `warden.exe` и выбери провайдера при первом запуске.

```powershell
# Или собрать из исходников (Go 1.25+, Python 3.11+)
just release
```

## Зачем это нужно

Есть задача — достаточно раздражающая, чтобы о ней думать, но недостаточно важная, чтобы писать скрипт. Warden таких не боится.

```
> найди пять самых больших файлов на рабочем столе и запакуй их в archive.zip
> пройдись по Downloads, удали всё старше 30 дней, PDF-ки оставь
> открой Chrome, зайди на GitHub, закрой все PR, которые висят больше 90 дней
> разберись почему упал последний CI и покажи нужные логи
> переименуй все скриншоты в папке по дате создания
```

Работает по всей машине — шелл, браузер, файловая система, процессы — и стримит каждое действие в чат по мере выполнения.

## Как это устроено

```
Ты пишешь задачу
      │
      ▼
  Скриншот → модель смотрит что на экране и думает
      │
      ▼
  Запускает команду / кликает / печатает / скроллит
      │
      ▼
  Скриншот снова → убеждается что сработало
      │
      ▼
  Следующий шаг (или готово)
```

Каждое действие видно в чате. **Ask-режим** делает паузу перед записью файлов и кликами мышью — ты остаёшься в контроле. **Auto-режим** даёт агенту работать без остановок, когда задаче доверяешь.

Переключаться между режимами — `Shift+Tab` в любой момент.

## Безопасность

Действия классифицирует код, а не модель — до того, как что-либо запустится.

| Уровень | Что | Поведение |
|---|---|---|
| Безопасно | Только чтение: файлы, скриншоты, `git status`, браузер | Выполняется сразу |
| Подтверждение | Запись, установка, мышь/клавиши, kill процессов | Пауза в Ask-режиме |
| Заблокировано | Рекурсивное удаление вне рабочей папки, обфусцированные команды, реестр | Всегда отклоняется |

Аварийная остановка: брось мышь в левый верхний угол — любое автоматическое действие прерывается.

## Память

Warden запоминает факты о твоих проектах и привычках между сессиями — хранится локально в SQLite, вставляется в контекст каждый раз.

```
/memory on          # включить
/memory off         # выключить
/memory status      # посмотреть что хранится
/memory clear       # стереть всё
```

## Скиллы

Переиспользуемые инструкции в обычном Markdown. Кинь `SKILL.md` в `.warden/skills/<name>/` и вызывай через `!`:

```
!<имя-скилла>       # запустить скилл
!<команда>          # запустить PowerShell-команду напрямую
!                   # список всех доступных скиллов
```

Скиллы живут в `.warden/skills/` (проектные) или `~/.warden/skills/` (глобальные).

## Конфигурация

Преднастройка через `~/.warden-config.json`, чтобы пропустить визард:

```json
{
  "model": "poolside/laguna-m.1:free",
  "api_url": "https://openrouter.ai/api/v1",
  "api_key": "sk-or-v1-..."
}
```

API-ключ хранится на диске в зашифрованном виде (DPAPI на Windows). Ключ и токен аутентификации передаются в бэкенд через stdin — никогда в переменных окружения. Каждый запрос к бэкенду аутентифицируется общим секретом, сгенерированным при старте.

Переменная окружения `WARDEN_MODEL` позволяет переопределить модель. Работает с Ollama (локально) или любым OpenAI-совместимым API.

## Управление

| Клавиша | Действие |
|---|---|
| `Enter` | Отправить сообщение |
| `Esc` | Прервать агента |
| `Esc × 2` | Принудительная остановка |
| `Shift+Tab` | Переключить Ask / Auto |
| `Tab` | Автодополнение команды или скилла |
| `↑` / `↓` | История ввода |
| `Scroll` | Прокрутка чата |
| `Ctrl+W` | Удалить последнее слово |
| `Ctrl+C` | Выход |

## Слэш-команды

| Команда | Действие |
|---|---|
| `/connect` | Настроить провайдера и модель |
| `/models` | Сменить модель на лету |
| `/clear` | Очистить чат, сбросить сессию |
| `/compact` | Сжать контекст чтобы освободить токены |
| `/memory` | Управление памятью |
| `/select` | Режим выделения текста |
| `/update` | Скачать и установить новую версию |
| `/verbose` | Показывать вызовы инструментов и диффы |

## Стек

| Слой | Технологии |
|---|---|
| TUI | Go 1.24+, bubbletea, lipgloss, glamour |
| Бэкенд | Python 3.11+, aiohttp |
| LLM | Ollama или любой OpenAI-совместимый API |
| Экран и ввод | pyautogui, Pillow |
| Браузер | Playwright |
| Поиск | duckduckgo-search |

Фронтенд и бэкенд полностью независимы, общаются по HTTP NDJSON. Можно заменить любую часть, не трогая другую.

## Сборка и тесты

```powershell
just build              # только Go TUI
just build-backend      # Python-бэкенд (требует PyInstaller)
just release            # всё вместе

just install            # зависимости Python
just test               # pytest с coverage
just test --no-cov -q   # быстрый прогон
just test-go            # Go-тесты
just lint-go            # go vet
just fmt-go             # проверка форматирования
```

## Если что-то не работает

| Симптом | Решение |
|---|---|
| `port 8765 is busy` | Уже запущен другой экземпляр — `taskkill /F /IM warden.exe` |
| `ollama is not responding` | Установи [Ollama](https://ollama.com) и запусти |
| Бэкенд сразу падает | Смотри `.warden/backend.err.log` |
