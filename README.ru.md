> **Опиши задачу. Не пиши скрипт.**

Warden — AI-агент, который живёт в твоём терминале и управляет компьютером. Говоришь что нужно сделать — он сам смотрит на экран, запускает команды, редактирует файлы, водит браузер и показывает каждый шаг в реальном времени.

Один бинарник, никакого Electron, не нужен аккаунт в облаке, никаких плагинов.

## Установка

[**⬇ Скачать для Windows x64**](https://github.com/elev1e1nSure/warden/releases/latest)

```powershell

```

## Зачем это нужно

Задачи — достаточно раздражающие, чтобы о них думать, но не настолько, чтобы писать скрипт.

```
> найди пять самых больших файлов на рабочем столе и запакуй их в archive.zip
> пройдись по Downloads, удали всё старше 30 дней, PDF-ки оставь
> открой Chrome, зайди на GitHub, закрой все PR, которые висят больше 90 дней
> разберись почему упал последний CI и покажи нужные логи
> переименуй все скриншоты в папке по дате создания
```

Работает по всей машине — шелл, браузер, файловая система, процессы.

---

## Как это устроено

```
Ты пишешь задачу
      │
      ▼
  Скриншот → модель смотрит что на экране и думает
      │
      ▼
  Запускает команду / кликает / печатает / скроллит
      │
      ▼
  Скриншот снова → убеждается что сработало
      │
      ▼
  Следующий шаг (или готово)
```

Каждое действие видно в чате. **Ask-режим** делает паузу перед записью файлов и кликами — ты остаёшься в контроле. **Auto-режим** даёт агенту работать без остановок, когда задаче доверяешь.

Переключаться между режимами — `Shift+Tab` в любой момент.

## Безопасность

Действия классифицирует Go-код, а не модель — до того, как что-либо запустится.

| Уровень | Что | Поведение |
|---|---|---|
| Безопасно | Только чтение: файлы, скриншоты, `git status`, браузер | Выполняется сразу |
| Подтверждение | Запись, установка, мышь/клавиши, kill процессов | Пауза в Ask-режиме |
| Заблокировано | Рекурсивное удаление вне рабочей папки, опасные пути, реестр | Всегда отклоняется |

Аварийная остановка: брось мышь в левый верхний угол — любое автоматическое действие прерывается.

## Память

Warden запоминает факты о твоих проектах и привычках между сессиями — хранится локально в SQLite, вставляется в контекст каждый раз.

```
/memory on          # включить
/memory off         # выключить
/memory status      # посмотреть что хранится
/memory clear       # стереть всё
```

## Скиллы

Переиспользуемые инструкции в обычном Markdown. Кинь `SKILL.md` в `.warden/skills/<name>/` и вызывай через `!`:

```
!<имя-скилла>       # запустить скилл
!<команда>          # запустить PowerShell-команду напрямую
!                   # список всех доступных скиллов
```

Скиллы живут в `.warden/skills/` (проектные) или `~/.warden/skills/` (глобальные).

## Конфигурация

Преднастройка через `~/.warden-config.json`, чтобы пропустить визард:

```json
{
  "model": "poolside/laguna-m.1:free",
  "api_url": "https://openrouter.ai/api/v1",
  "api_key": "sk-or-v1-..."
}
```

API-ключ хранится на диске в зашифрованном виде (DPAPI на Windows). Ключ и токен аутентификации передаются в бэкенд через stdin — никогда в переменных окружения. Каждый запрос аутентифицируется общим секретом, сгенерированным при старте.

Переменная окружения `WARDEN_MODEL` позволяет переопределить модель. Работает с Ollama (локально) или любым OpenAI-совместимым API.

## Управление

| Клавиша | Действие |
|---|---|
| `Enter` | Отправить сообщение |
| `Esc` | Прервать агента |
| `Esc × 2` | Принудительная остановка |
| `Shift+Tab` | Переключить Ask / Auto |
| `Tab` | Автодополнение команды или скилла |
| `↑` / `↓` | История ввода |
| `Scroll` | Прокрутка чата |
| `Ctrl+W` | Удалить последнее слово |
| `Ctrl+C` | Выход |

## Слэш-команды

| Команда | Действие |
|---|---|
| `/connect` | Настроить провайдера и модель |
| `/models` | Сменить модель на лету |
| `/clear` | Очистить чат, сбросить сессию |
| `/compact` | Сжать контекст чтобы освободить токены |
| `/memory` | Управление памятью |
| `/select` | Режим выделения текста |
| `/update` | Скачать и установить последний релиз |

## Архитектура

Один Go-бинарник — никакого отдельного бэкенда.

| Слой | Технология |
|---|---|
| TUI | bubbletea, lipgloss, glamour |
| Цикл агента | Нативный Go — скриншот → LLM → выполнение |
| LLM | Ollama или любой OpenAI-совместимый API |
| Экран и ввод | Win32 API через `kbinani/screenshot`, SendInput |
| Браузер | Playwright for Go (`mxschmitt/playwright-go`) |
| Поиск | DuckDuckGo instant answer API |
| Память | SQLite через `modernc.org/sqlite` |
| Безопасность | Детерминированно — `agent/safety/policy.go` |

Агент работает целиком внутри Go-процесса. Инструменты — встроенные Go-реализации: PowerShell, файловые операции, управление браузером, мышь/клавиатура, OCR, поиск по изображению и другие.

## Сборка и тесты

```powershell
just build              # Собрать warden.exe
just test               # Запустить Go-тесты
just test -v ./agent    # Тесты конкретного пакета
just test-cov           # С покрытием
just lint               # go vet
just fmt                # Проверить форматирование
just fmt-write          # Исправить форматирование
```

В justfile нет Python-целей — проект полностью на Go.

## Проблемы

| Симптом | Решение |
|---|---|
| `port 8765 is busy` | Ещё один экземпляр запущен — `taskkill /F /IM warden.exe` |
| `ollama is not responding` | Установи [Ollama](https://ollama.com) и запусти |
| Бэкенд сразу закрывается | Проверь `.warden/backend.err.log` |
